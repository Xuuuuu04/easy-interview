# EasyInterview (易面) 交互优化与视觉升级计划

针对您的需求，我将从用户体验 (UX)、功能增强和视觉一致性三个方面进行全面升级。

## 1. 交互体验优化 (UX Improvements)
*   **面试计划加载反馈**: 
    *   在 `submitContext` 函数中，虽然已有加载提示，我将进一步增强其视觉效果，使用全屏模态框 (Modal) 替代当前的文字提示，并强制禁止任何点击操作，确保用户“耐心等待”。
    *   **新增机制**: 在点击“开始”后，直到进入面试房间前，界面将被锁定，防止重复提交。
*   **首条消息生成**: 
    *   确认点击开始后，系统会先生成开场白（Opening Line），再允许用户进入房间。

## 2. 功能增强 (Feature Upgrades)
*   **面试官/音色选择**:
    *   **位置**: 在面试界面底部的控制栏（麦克风按钮旁）添加一个风格化的下拉菜单。
    *   **功能**: 列出所有可用 TTS 音色（如 "Anna", "David" 等），支持实时切换，让用户自定义面试官的声音风格。
*   **警告系统重构**:
    *   **当前问题**: 警告信息挤占了右侧计划列表的空间。
    *   **解决方案**: 将 `<div id="cheat-alert">` 移至页面顶部中央，设计为类似“系统弹窗”的悬浮 Toast 样式。
    *   **行为**: 警告触发时从顶部滑入，显示 5 秒后自动收起，不再破坏页面布局。

## 3. 视觉与设计语言统一 (Design System Unification)
*   **去“青”化 (De-Cyanization)**: 
    *   全面排查 `app.js` 和 `index.html`，将所有残留的 `cyan` (青色) 样式替换为 `purple` (紫色) 或 `fuchsia` (洋红)，确保与 "EasyInterview" 的新品牌色一致。
    *   **Canvas 重绘**: 更新音频波形图和人脸识别框的绘图颜色，使其与 UI 主题融合。
*   **字体与质感**:
    *   增强文字的“外发光” (Glow) 效果，使用统一的紫色光晕。
    *   优化卡片背景的透明度和模糊度 (Backdrop Blur)，提升层级感。
